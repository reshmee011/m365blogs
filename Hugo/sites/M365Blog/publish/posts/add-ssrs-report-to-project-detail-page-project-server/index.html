Add SSRS report to Project Detail Page Project Server

February 23, 2016

Often clients want to be able to drill down to SSRS reports with the project pre selected when they are viewing project information from project detail pages. This post describes how to embed SSRS report in a project detail page.

Add SSRS Report to Project Detail Page in Project Server

1. Navigate to a PDP (Project Detail Page) , e.g. Schedule PDP. 2. Edit Page 3. Click “Add a Web Part”. Select the “SQL Server Reporting Services Report Viewer” web part. 4. Click “Add a Web Part”. Select the “SQL Server Reporting Services Report Viewer” web part. 5. Click “Click here to open the tool pane”. 6. Click on the “…” button to select a SSRS report and click apply. In the example below the “Project Overall Status” has been selected with one parameter capable of accepting ProjectUID. 7. Click “Add a Web Part” to add “Query String (URL) filter”. 8. Click “Open the tool pane” to configure “Query String (URL) filter”. 9. Add “projuid” into the “Query String Parameter Name” property. 10. Click on the “Arrow Down” on the webpart and click on “Connections”-”Send Filter Values To” - . 11. Click on the “Arrow Down” on the webpart and click on “Connections”-”Send Filter Values To” - . Pick the parameter defined in the SSRS report into “Filtered Parameter”. 12. Click on “Stop Editing”. 13. The report is rendered displaying only information of the project
