Changing Date Parameterer to en-GB SSRS 2012 SharePoint Integrated mode

I have seen the issues several times where date/time parameters used to filter date in SSRS reports are switched to en-US despite the language of the reports is set to en-GB. If dates like 24/12/2015 or 15/12/2015 are picked , the report fails to load due to invalid date format. I came across the article that describes a fix. http://www.tachytelic.net/2013/01/changing-the-format-of-date-parameters-on-a-sql-server-2012-reporting-services-report-that-is-sharepoint-integrated/ Hard code the UICulture value in RSViewerPage.aspx, found in C:\Program Files\Common Files\Microsoft Shared\Web Server Extensions\15\TEMPLATE\LAYOUTS\ReportServer The modified code will look like: <%@ Page language=“C#” Codebehind=“RSViewerPage.
